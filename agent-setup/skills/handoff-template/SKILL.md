## What I do
- Produce a repo-ready `HANDOFF.md` at the project root.
- Keep it short, specific, and verifiable.

## When to use me
Use when starting a new task/PR or when handing work off between agents/humans.

## Output rules
- Use the template sections: Goal, Non-goals, Constraints, Acceptance gates, Files touched, How to verify, Rollback plan, Open questions/risks.
- Prefer concrete commands in "How to verify".
